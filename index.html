Android Studio使用Logcat打印日志
发表于2022-01-17|Android学习
常用日志方法方法优先级从高到低： Log.e(String tag, String message)（error） 错误, tag 表示打印时的标签，用于过滤信息； message一般写为msg 表示打印内容 Log.w(String tag, String message)（warning） 警告 Log.i(String tag, String message)（info） 信息 Log.d(String tag, String message)（debug） 调试 Log.v(String tag, String message)（verbose 冗长的，啰嗦的） 无关紧要 注意： 除开发期间，其他任何时候都不应将详细日志编译到您的应用中。虽然会编译调试日志，但会在运行时将其去掉，而错误、警告和信息日志会始终保留。 Debug Debug是跟踪程序流程的一种模式，可以通过在代码处设置断点，再利用Debug窗口查看，所谓断点是在代码行加入停止点，当程序执行到该行时会暂停，开发者可以从中查看到此行的变量、数值和内容等。
